# GitHub CI Workflow Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a GitHub Actions CI workflow that runs lint, typecheck, test, and build in parallel on every PR and push to main.

**Architecture:** Single `.github/workflows/ci.yml` with 4 independent parallel jobs sharing identical setup (Node 20, pnpm 9, Bun). All tests are fully mocked — no database or Redis required.

**Tech Stack:** GitHub Actions, pnpm 9, Bun, Turborepo, Node 20

---

### Task 1: Create the CI workflow file

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Create directory**

```bash
mkdir -p .github/workflows
```

**Step 2: Write the workflow file**

Create `.github/workflows/ci.yml` with the following content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  lint:
    name: Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - uses: oven-sh/setup-bun@v2

      - run: pnpm install --frozen-lockfile

      - run: pnpm lint

  typecheck:
    name: Typecheck
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - uses: oven-sh/setup-bun@v2

      - run: pnpm install --frozen-lockfile

      - run: pnpm typecheck

  test:
    name: Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - uses: oven-sh/setup-bun@v2

      - run: pnpm install --frozen-lockfile

      - run: pnpm test

  build:
    name: Build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - uses: oven-sh/setup-bun@v2

      - run: pnpm install --frozen-lockfile

      - run: pnpm build
```

Key design decisions:
- `pnpm/action-setup@v4` reads `packageManager` field from `package.json` automatically (no version param needed)
- `actions/setup-node@v4` with `cache: pnpm` handles pnpm store caching
- `oven-sh/setup-bun@v2` installs latest Bun (needed for api and shared tests/build)
- `concurrency` cancels in-progress runs when a new push arrives on the same branch
- No `--frozen-lockfile` on build since `pnpm install` already handles it
- Jobs are fully independent — one failure does not cancel others

**Step 3: Verify YAML syntax locally**

Run: `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))" && echo "YAML valid"`
Expected: `YAML valid`

**Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions workflow for lint, typecheck, test, and build"
```

### Task 2: Verify CI passes on GitHub

**Step 1: Push branch and create PR**

```bash
git push -u origin HEAD
gh pr create --title "ci: add GitHub Actions CI workflow" --body "..."
```

**Step 2: Monitor CI run**

```bash
gh run list --limit 1
gh run watch
```

Expected: All 4 jobs (Lint, Typecheck, Test, Build) pass green.

**Step 3: If any job fails, read logs and fix**

```bash
gh run view <run-id> --log-failed
```

Fix issues, push, and re-check until all jobs pass.
